## Usage

Activate this skill by:
- Using the `/accelint-ts-best-practices` command with a file or directory path
- Saying "review TS/JS code", "audit TypeScript", "check for type errors", "apply best practices"
- Mentioning "any type", "enum", "null returns", "defensive programming", or "code correctness"
- Working with `.ts`, `.tsx`, `.js`, `.jsx` files

## What It Covers

**General Best Practices** - Naming conventions, function design, control flow patterns, state management, return values, DRY principle

**TypeScript Patterns** - Avoiding `any`, replacing `enum` with `as const`, preferring `type` over `interface`, bundler-friendly paths

**Safety** - Input validation with schemas, assertions with values, explicit error handling, error messages, bounded iteration

**Anti-Patterns** - Critical mistakes that degrade code quality (never use `any`, never use `enum`, never return `null`, never create unbounded loops)

## Examples

### Example: Fixing Any Types

```bash
# Command
/accelint-ts-best-practices src/api/client.ts

# Issue Found
âťŚ function processData(data: any) {
  return data.results.map(item => item.id);
}

# Fix Applied
âś… function processData<T extends { results: Array<{ id: string }> }>(data: T) {

```

**Why:** The `any` type disables type checking and propagates through the codebase. Generics preserve type safety while maintaining flexibility.

### Example: Replacing Enums

```bash
# Issue Found
âťŚ enum Status {
  Active = "active",
  Inactive = "inactive"
}

# Fix Applied
âś… const Status = {
  Active: "active",
  Inactive: "inactive"
} as const;

type Status = typeof Status[keyof typeof Status];
```

**Why:** The `enum` keyword generates 5+ lines of runtime code. The `as const` object provides zero-cost type-safe constants with no JavaScript output.

### Example: Zero-Value Returns

```bash
# Issue Found
âťŚ function getActiveUsers(users: User[]): User[] | null {
  if (!users.length) return null;
  return users.filter(u => u.active);
}

# Fix Applied
âś… function getActiveUsers(users: User[]): User[] {
  if (!users.length) return [];
  return users.filter(u => u.active);
}
```

**Why:** Returning `null` forces defensive null checks throughout the codebase. Returning empty arrays eliminates these checks and enables method chaining.

### Example: Bounded Iteration

```bash
# Issue Found
âťŚ while (queue.length > 0) {
  const item = queue.shift();
  processItem(item);
}

# Fix Applied
âś… const MAX_ITERATIONS = 10000;
let iterations = 0;
while (queue.length > 0 && iterations < MAX_ITERATIONS) {
  const item = queue.shift();
  processItem(item);
  iterations++;
}
if (iterations >= MAX_ITERATIONS) {
  throw new Error(`Queue processing exceeded ${MAX_ITERATIONS} iterations`);
}
```

**Why:** Unbounded loops can cause runaway resource consumption and system crashes if the queue grows faster than it's processed.

### Critical Anti-Patterns

**Never do these** - they appear frequently but degrade quality:

- Never use `any` type (use `unknown` or generics)
- Never use `enum` keyword (use `as const` objects)
- Never use `interface` for simple types (use `type` instead)
- Never mutate function parameters (creates hidden side effects)
- Never return `null` or `undefined` (return zero values)
- Never create unbounded loops or queues (set explicit limits)
