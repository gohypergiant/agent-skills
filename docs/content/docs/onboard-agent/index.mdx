## What It Helps You Do

Use this skill to create project-specific agent behavior instructions that define how an AI coding agent should work across all interactions in your project.

It is especially useful when you need to:
- Generate a complete AGENTS.md or CLAUDE.md through a guided interview
- Define agent identity, communication style, and workflow procedures
- Establish decision-making heuristics and behavioral guardrails
- Import and restructure existing agent instructions
- Refresh stale agent configuration when your project evolves

The skill produces the **behavior layer** of your agent instruction stack—defining WHO the agent is and HOW it works—while keeping project facts (stack, architecture, coding standards) separate in `openspec/config.yaml`.

Activate this skill by:
- Using the `/accelint-onboard-agents` command
- Saying "create AGENTS.md", "onboard agent", "configure claude code"
- Mentioning "agent behavior", "agent instructions", or "agent guardrails"

## When to Use

Use this skill when:
- Users mention "AGENTS.md", "CLAUDE.md", "agent behavior", "agent instructions", or "agent config"
- Setting up agent rules for a new project
- Asking how to configure Claude Code or other AI agents for a project
- Onboarding a project to agent-driven development
- Working with agent workflows or guardrails

## How It Works

The skill runs an adaptive interview that adjusts based on what already exists in your project:

- **Create mode** — Full interview from scratch for new projects
- **Import mode** — Maps existing content onto a standardized template, flags content that belongs in other files
- **Refresh mode** — Detects codebase drift since last update and fills gaps with targeted questions

### Intelligent Discovery Process

Before asking questions, the skill scans the repo for conventions around version control, CI/CD, testing, security, and development workflow. It then asks only about the gaps it could not infer and shows inferred values with source annotations in the preview.

### Separation of Concerns

The skill keeps behavior instructions in `AGENTS.md` or `CLAUDE.md` and project facts in `openspec/config.yaml`. If you provide stack facts during the interview, it redirects those details to the OpenSpec config layer instead.

### Monorepo Support

In a monorepo, package-level files reference the root agent instructions where possible instead of duplicating shared guidance.

## What You Get

A complete `AGENTS.md` or `CLAUDE.md` draft covering role and identity, communication style, workflows, decision heuristics, tool preferences, guardrails, and links to related project docs.

Sections that still need human input are marked with `<!-- TODO: fill in -->` rather than being omitted.

## Good to Know

> **Good to know:** The skill shows a labeled preview of the full AGENTS.md before writing anything. Inferred values carry source comments so you can see exactly where each convention came from.

> **Good to know:** For monorepos with a root-level AGENTS.md, package-level files should reference it and add only package-specific overrides. Repeated instructions across root and package files inflate context on every agent invocation.

> **Good to know:** If you have an existing AGENTS.md with unrecognized structure, the skill offers three options: restructure it onto the standard template, append new sections below the existing content, or run a dry run to preview what would be generated.

## Prerequisites

- Git repository (for computing source_sha and detecting configuration)
- Write access to create or modify AGENTS.md or CLAUDE.md at project root
