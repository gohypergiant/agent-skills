## What It Helps You Do

Use this skill to audit JavaScript/TypeScript applications for security vulnerabilities and apply OWASP Top 10 security patterns.

It is especially useful when you need to:
- Conduct security audits before deployment
- Identify and fix authentication, authorization, or injection vulnerabilities
- Review code for hardcoded secrets, insecure storage, or data exposure
- Add security controls around APIs, file uploads, and user input
- Improve alignment with OWASP guidance

Security guidance is framework-agnostic. The examples illustrate the pattern, but the implementation should fit your stack.

## When to Use

Use this skill when:
- Users say "audit security", "check for vulnerabilities", "security review", "implement authentication", or "secure this code"
- Adding authentication, API endpoints, file uploads, or handling user input
- Working with secrets, credentials, or sensitive data
- Implementing payment features or blockchain integrations
- Conducting pre-deployment security checks

## How It Works

The skill works in two modes:

- **Audit mode** produces a structured security review so you can decide what to fix first.
- **Implementation mode** applies security fixes inline when security issues surface during feature work.

In both modes, it follows the same basic sequence:
1. Discover vulnerabilities in the codebase
2. Categorize them by OWASP area and severity
3. Apply the relevant remediation pattern
4. Verify that fixes hold and do not introduce new issues

## Good to Know

> **Good to know:** This skill focuses on security vulnerabilities and OWASP-style remediation patterns. For general TypeScript quality, performance, or documentation work, use the corresponding accelint-ts skills.

> **Good to know:** The guidance becomes more prescriptive as severity increases. Critical and high-severity issues get less flexibility than lower-risk findings.

> **Good to know:** When security and convenience conflict, the skill prioritizes security for critical vulnerabilities.

## Prerequisites

- JavaScript/TypeScript codebase
- Node.js runtime (the skill works with any package manager: npm, yarn, pnpm, bun)
- Git repository (for computing source SHA hashes)
- Existing test suite (recommended for validating security fixes)

## What You Get

**In audit mode:**
- A structured security review with OWASP categories and severity levels
- Prioritized findings ready for remediation
- Recommended patterns for each vulnerability type

**In implementation mode:**
- Security fixes applied directly to code
- Updated code that follows the recommended remediation pattern

**In both modes:**
- Vulnerability categorization by OWASP area
- Severity assessment
- Actionable remediation guidance

## Examples

### Example: Auditing Authentication Implementation

```bash
# Command
/accelint-security-best-practices src/auth/

# Output: Structured audit report
Phase 1: Discovered 3 vulnerabilities
- JWT token stored in localStorage (High)
- Missing rate limiting on /api/login (High)
- No session expiration (Medium)

Phase 2: Categorized by OWASP
- A07: Authentication Failures (2 issues)
- A04: Insecure Design (1 issue)

Phase 3: Remediation patterns loaded
- authentication.md: httpOnly cookie patterns
- rate-limiting.md: login endpoint protection
```

### Example: Fixing SQL Injection Vulnerability

```typescript
// Before: SQL Injection vulnerability
const query = `SELECT * FROM users WHERE email = '${email}'`;
const user = await db.query(query);

// After: Parameterized query prevents injection
// Security: injection-prevention.md - parameterized queries
const user = await db.query(
  'SELECT * FROM users WHERE email = $1',
  [email]
);
```

### Example: Securing API Endpoints

```typescript
// Before: Missing authentication and authorization
app.get('/api/user/:id', async (req, res) => {
  const user = await db.query('SELECT * FROM users WHERE id = $1', [req.params.id]);
  res.json(user);
});

// After: Defense in depth with auth + authorization
// Security: authentication.md + authorization.md
app.get('/api/user/:id', 
  authenticate, // Verify user is logged in
  async (req, res) => {
    const requestedId = req.params.id;
    // Verify user owns this resource
    if (req.user.id !== requestedId && !req.user.isAdmin) {
      return res.status(403).json({ error: 'Forbidden' });
    }
    const user = await db.query('SELECT * FROM users WHERE id = $1', [requestedId]);
    res.json(user);
  }
);
```

## Limits

- This skill audits code for security vulnerabilities but does not perform runtime penetration testing or network security analysis
- Dependency vulnerability detection relies on identifying outdated packages—it does not scan for zero-day vulnerabilities
- The skill provides security patterns for common frameworks but may require adaptation for custom authentication systems or proprietary frameworks
- For infrastructure security (cloud configurations, network policies, container security), use specialized infrastructure security tools
