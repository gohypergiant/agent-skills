## Usage

Activate this skill by:
- Using the `/accelint-prompt-manager` command
- Saying "optimize this prompt", "improve my prompt", "make this clearer", "help me figure out what to ask"
- Mentioning vague goals like "make this better", "I don't know where to start", "not sure what this means"
- Expressing unclear requirements like "should look professional", "explain this clearly", "whatever works best"
- Asking about prompt frameworks like CO-STAR, RISEN, or RODES

```bash
# Optimize a specific prompt
/accelint-prompt-manager "make the documentation comprehensive"

# Get help with unclear requests
/accelint-prompt-manager "improve our X"

# Learn about prompt optimization
/accelint-prompt-manager "review my instructions"
```

Once activated, the skill runs a 4-phase workflow to transform your prompt:

1. **Intake &amp; Assessment** — Understands your intent, skill level, task complexity, and execution context
2. **Pattern Detection** — Identifies credit-killing patterns, ambiguities, trade-offs, and missing context
3. **Framework Selection &amp; Optimization** — Applies CO-STAR, RISEN, or RODES silently with safe techniques
4. **Validation &amp; Handoff** — Quality-checks and delivers the optimized prompt directly to you

**Critical:** This skill optimizes prompts—it does NOT execute the tasks they describe. You'll receive a clear, actionable prompt that you (or Claude) can then execute.

## When to Use

Use this skill when:
- Users provide vague, underspecified, or unclear requests
- Requests express vague goals ("make this better", "improve our X", "I don't know where to start")
- Success criteria are undefined ("should look professional", "make it convincing", missing constraints/audience/format)
- Communication is unclear ("how do I explain this", "my team gets confused", "help me figure out what to ask")
- Requirements are ambiguous ("analyze the data" without specifics, "make it more robust" without defining robustness)
- Users want to learn about meta-prompting ("optimize this prompt", learning frameworks like CO-STAR/RISEN/RODES)
- Non-technical users need help structuring requests
- Any situation where the request needs refinement before execution can begin

**When in doubt about whether a request is clear enough—trigger this skill.**

## What This Skill Does

**Role &amp; Output**
- Produces optimized prompts as the sole artifact
- Does NOT execute tasks—only optimizes the prompts that describe them
- Does NOT research external resources—works only with provided input text
- Always delivers optimized prompt directly in markdown code block first
- Offers post-delivery options to save to file or copy to clipboard

**Optimization Approach**
- Detects credit-killing anti-patterns (fabrication techniques, inappropriate CoT, framework pollution)
- Identifies ambiguities and presents interpretation options with trade-offs
- Applies appropriate framework (CO-STAR/RISEN/RODES) silently without exposing methodology
- Uses safe techniques: specificity injection, constraint addition, context positioning, pronoun elimination
- Validates against measurable success criteria

**Context-Aware**
- Calibrates to user skill level (newcomer, intermediate, expert)
- Assesses task complexity (simple/moderate/complex)
- Recommends plan mode for complex tasks (&gt;3 interdependent decisions OR &gt;5 phases)
- Adapts to execution context (interactive, API, batch, system prompt)

## Anti-Patterns

> **Good to know:** This skill focuses on prompt optimization, not task execution. If you provide a prompt like "Create a skill that uses GitHub APIs", the skill will optimize that PROMPT TEXT—it won't fetch GitHub docs or research APIs. The input text is the raw material to optimize, not a task to execute.

> **Good to know:** Avoid these credit-killing patterns that degrade prompt quality: embedding fabrication techniques (MoE, ToT, GoT) in single-prompt execution, adding Chain-of-Thought instructions to reasoning-native models like Claude 4.5+, naming frameworks in output ("Using CO-STAR..."), optimizing prompts without considering execution context, using vague success criteria, skipping constraints for creative tasks, front-loading all context in long prompts, using ambiguous pronouns in multi-step instructions.

> **Good to know:** The skill recommends plan mode for complex tasks (&gt;3 interdependent decisions OR &gt;5 sequential phases). If you explicitly decline plan mode, the skill continues but notes the complexity. For simple tasks (&lt;3 steps, single objective), it proceeds directly without the recommendation.

> **Good to know:** Framework selection is automatic based on task type: CO-STAR for format-driven tasks (structured output needs), RISEN for process-driven tasks (multi-step procedures), RODES for example-driven tasks (needs style matching). The framework is applied silently—you won't see methodology labels in the output.

## Key Patterns

### Credit-Killing Pattern Detection

The skill scans for 8 anti-patterns that undermine prompt effectiveness:

1. **Fabrication Techniques** — MoE/ToT/GoT make Claude invent fake personas instead of deepening reasoning
2. **Inappropriate CoT** — Claude 4.5+ has native extended thinking; "think step-by-step" degrades quality
3. **Framework Name Pollution** — Exposing methodology ("Using CO-STAR...") in output
4. **Context-Free Optimization** — Ignoring execution environment differences (Claude Code vs ChatGPT vs API)
5. **Vague Success Criteria** — "Better", "comprehensive", "clean" lack measurability
6. **Missing Creative Constraints** — Without boundaries, creative tasks produce inconsistent results
7. **Lost-in-the-Middle** — Critical info buried in long prompts where attention weakens
8. **Ambiguous Pronouns** — "It/this/that" become unclear across multiple steps

If 3+ patterns are detected, the skill loads detailed examples from `references/credit-killing-patterns.md`.

### Ambiguity Resolution

Common vague terms and their interpretation options:

- **"Comprehensive"** → All edge cases [+time] vs common scenarios [balanced] vs overview [+speed]
- **"Fast"** → Response time, development time, or execution time?
- **"Simple"** → Minimal code, easy to understand, or few dependencies?
- **"Clean"** → Follows standards, minimal complexity, or well-documented?

The skill presents 2-3 interpretation options with implications and lets you decide the trade-off.

### Safe Optimization Techniques

Five proven methods that improve quality without risk:

1. **Specificity Injection** — Replace vague terms with concrete criteria
2. **Constraint Addition** — Define boundaries for creative freedom
3. **Context Positioning** — Place critical info at start/end, not middle
4. **Pronoun Elimination** — Replace "it/this/that" with specific nouns
5. **Success Criteria Definition** — Pin to measurable outcomes

### Complexity Assessment

Task complexity determines optimization approach:

- **Simple:** Single objective, &lt;3 steps, no ambiguity → Execute directly
- **Moderate:** Some ambiguity, 3-5 steps, few dependencies → Proceed with monitoring
- **Complex:** &gt;3 interdependent decisions OR &gt;5 sequential phases → Recommend plan mode

### Framework Selection Logic

Automatic framework choice based on task type:

- **CO-STAR (Format-driven)** — Structured output, specific format needs
  - Components: Context, Objective, Style, Tone, Audience, Response format
- **RISEN (Process-driven)** — Multi-step procedures, workflows
  - Components: Role, Instructions, Steps, End goal, Narrowing
- **RODES (Example-driven)** — Needs examples for clarity, style matching
  - Components: Role, Objective, Details, Examples, Sense check

Frameworks are applied silently—methodology is never exposed in the optimized output.

## Workflow Phases

The skill follows a systematic 4-phase process with a progress checklist:

```markdown
- [ ] Phase 1: Intake &amp; Assessment
- [ ] Phase 2: Pattern Detection
- [ ] Phase 3: Framework Selection &amp; Optimization
- [ ] Phase 4: Validation &amp; Handoff
```

### Phase 1: Intake &amp; Assessment

Understands your request:
- Extracts core intent from vague descriptions
- Assesses skill level (newcomer/intermediate/expert) from language and terminology
- Detects task complexity by counting decision points, dependencies, and phases
- Identifies execution context (interactive chat, API call, system prompt)

For complex tasks, recommends plan mode before proceeding.

### Phase 2: Pattern Detection

Identifies issues:
- Scans for credit-killing patterns (loads reference if 3+ detected)
- Flags ambiguous terms with interpretation options
- Identifies trade-offs (speed vs thoroughness, flexibility vs consistency)
- Assesses missing context (audience, success criteria, constraints, format)

Output format adapts to skill level (explanatory for newcomers, terse for experts).

### Phase 3: Framework Selection &amp; Optimization

Applies appropriate structure:
- Selects CO-STAR, RISEN, or RODES based on task type
- Routes user intent through framework structure silently
- Applies safe techniques (specificity, constraints, positioning, pronouns, criteria)
- Addresses all flagged issues from Phase 2
- Formats for execution context (conversational, API, system prompt, tool integration)

### Phase 4: Validation &amp; Handoff

Quality-checks and delivers:
- Runs 7 validation checks (ambiguities resolved, criteria measurable, constraints specified, etc.)
- Flags remaining ambiguities requiring user decisions
- Recommends execution mode (direct, monitored, or plan mode)
- **Delivers optimized prompt directly in markdown code block** (easy copying)
- Offers post-delivery options: save to file, copy to clipboard, or both
- Offers iteration: refine specific aspects or adjust for different contexts

**Critical:** The skill never offers to execute the task—only to optimize the prompt.

## Freedom Calibration

How closely to follow the optimization guidelines:

| Task Type | Freedom Level | Guidance |
|-----------|---------------|----------|
| **Meta-prompts / System prompts** | Low | Follow framework structures exactly—these define behavior for other prompts |
| **Production optimization** | Medium | Apply frameworks with examples—balance consistency with context-specific needs |
| **Creative prompt design** | High | Use principles and anti-patterns as guardrails—adapt freely to creative vision |

Higher fragility (left) = stricter adherence. Lower fragility (right) = more adaptation freedom.
