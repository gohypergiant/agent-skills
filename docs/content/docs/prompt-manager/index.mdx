## What It Helps You Do

Use this skill to clarify and structure unclear requests before attempting to execute them. It's particularly valuable when you're not quite sure how to articulate what you need, or when your initial request lacks the specificity needed for good results.

The skill helps you:
- Convert vague goals into concrete, actionable prompts with measurable success criteria
- Identify and resolve ambiguities that would lead to misaligned results
- Choose appropriate prompt frameworks (CO-STAR, RISEN, RODES) based on task type
- Remove patterns that waste tokens or degrade output quality
- Calibrate prompts to specific execution contexts (Claude Code, API calls, system prompts)

**Important:** This skill optimizes prompts—it doesn't execute the tasks they describe. You'll receive a refined prompt ready to use, not a completed task.

## When to Use

Activate this skill when:
- Users express vague goals: "make this better", "improve our X", "I don't know where to start"
- Success criteria are undefined: "should look professional", "explain clearly", missing constraints
- Requirements have multiple valid interpretations: "analyze the data" without specifying what to look for
- You're working on prompt engineering: "optimize this prompt", "review my instructions"
- Communication feels unclear: "how do I explain this", "help me figure out what to ask"

You can invoke it with:
- The `/accelint-prompt-manager` command
- Phrases like "help me write a better prompt for X"
- Keywords like "optimize prompt", "clarify request", "structure this"

## How It Works

The skill follows a four-phase workflow:

### Phase 1: Intake & Assessment
- Extracts the core intent from your request
- Calibrates to your skill level (newcomer, intermediate, expert)
- Assesses task complexity (simple, moderate, complex)
- Identifies execution context (where and how the prompt will run)

For complex tasks involving multiple interdependent decisions or more than five sequential phases, the skill recommends using plan mode before proceeding.

### Phase 2: Pattern Detection
- Scans for credit-killing patterns that waste tokens or degrade output quality
- Flags ambiguous terms that could be interpreted multiple ways
- Identifies trade-offs between competing goals (speed vs thoroughness, flexibility vs consistency)
- Surfaces missing context (audience, success criteria, constraints, format requirements)

The skill provides 2-3 interpretation options for each ambiguity with clear implications.

### Phase 3: Framework Selection & Optimization
Applies the appropriate prompt framework silently based on task type:
- **CO-STAR:** For structured output and specific format needs (format-driven)
- **RISEN:** For multi-step procedures and workflows (process-driven)
- **RODES:** When examples would clarify expectations or style (example-driven)

The framework is applied transparently—you won't see methodology labels in the output.

Uses safe optimization techniques:
- Replaces vague terms with concrete criteria
- Adds constraints to define creative boundaries
- Positions critical information at start/end (not lost-in-middle)
- Eliminates ambiguous pronouns in multi-step instructions
- Pins success criteria to measurable outcomes

### Phase 4: Validation & Handoff
- Runs quality checks on the optimized prompt
- Flags any remaining ambiguities that need your decision
- Recommends execution mode (direct, monitored, or plan mode)
- Delivers the optimized prompt directly in a markdown code block for easy copying
- Offers to save to file and/or copy to clipboard

After delivery, the skill offers to iterate if you want to refine specific aspects.

## What You Get

The primary deliverable is an optimized prompt presented directly in your conversation, formatted in a markdown code block for easy copying.

After delivering the prompt, you'll be offered optional post-delivery actions:
- Save the prompt to a markdown file (with path suggestions)
- Copy the prompt to your clipboard
- Both save and copy

For newcomers, you'll also see a before/after comparison explaining key changes. For experts, you'll receive concise optimization notes.

## Good to Know

> **Good to know:** The skill adapts its communication style to your skill level. Newcomers receive explanations of what's being detected and why it matters. Experts get pattern names and line numbers directly.

> **Good to know:** This skill never fetches external resources or researches topics. If you provide a prompt mentioning "GitHub APIs" or other external systems, the skill optimizes the prompt text itself—it doesn't investigate those systems.

> **Good to know:** Optimization strategies are model-specific. Claude 4.5+ uses extended thinking natively and doesn't benefit from explicit "think step by step" instructions, while older models might. The skill considers target model capabilities.

## Examples

### Example: Converting a Vague Request into an Actionable Prompt

```bash
# Input
"make our documentation better"

# After optimization
Context: You're improving technical documentation for a Node.js library's README.
Objective: Restructure documentation to reduce new user onboarding time from 30 minutes to under 10 minutes.
Success Criteria:
- Quick Start section completable in &lt;3 minutes
- API reference with copy-paste examples for top 5 use cases
- Troubleshooting section covering 3 most common setup errors
Target Audience: JavaScript developers with npm experience but no prior knowledge of this library.
Constraints:
- Keep README under 500 lines
- Include runnable code examples, not pseudocode
- Link to full docs for advanced topics rather than embedding everything
Format: Markdown with clear section headers and table of contents.
```

### Example: Optimizing an Ambiguous Prompt

```bash
# Input
"Analyze this codebase and tell me what's wrong"

# After optimization
Role: Senior code reviewer conducting a security and performance audit.
Objective: Identify critical issues in [codebase] that would block production deployment.
Focus Areas (prioritized):
1. Security vulnerabilities (SQL injection, XSS, exposed secrets)
2. Performance bottlenecks (O(n²) algorithms, N+1 queries, memory leaks)
3. Error handling gaps (unhandled rejections, missing validation)
Scope: Scan src/ directory only, exclude test files and vendor code.
Output Format:
- Executive summary with risk levels (Critical/High/Medium/Low)
- Per-issue: location, impact, remediation steps
- Prioritized fix list with estimated effort
Constraints: Complete review within current token budget; flag but don't fix issues yourself.
```

## Good to Know

> **Good to know:** This skill optimizes prompts rather than completing the underlying task. If you need the work done, use the refined prompt with Claude or another model afterward.

> **Good to know:** If it's unclear whether you need prompt optimization or direct task execution, the skill will pause to clarify before proceeding.

> **Good to know:** Optimization patterns are model-specific. Claude, GPT, and other model families respond differently to the same prompt structure.
