## Key Anti-Patterns

The skill emphasizes critical mistakes that lead to security issues, performance problems, or data corruption:

**Security &amp; Data Leakage**
- NEVER use a singleton QueryClient on the server - causes data leakage between users and race conditions
- Each server request must get its own isolated QueryClient instance

**State Management**
- NEVER synchronize query data to useState - background refetches make local copies stale immediately
- Use query data directly or derive with useMemo

**Performance**
- NEVER put queries inside list item components - creates N observers and N network requests for N items
- NEVER use unstable query keys (Date.now(), unsorted arrays, non-deterministic objects)
- NEVER skip enabled guards for dependent queries - fires requests with undefined parameters

**Mutations**
- NEVER use optimistic updates for high-stakes operations (life-critical, audit trails, external triggers)
- NEVER skip onSettled in optimistic updates - it's your cleanup guarantee even if onError throws
- NEVER assume cache invalidation is synchronous - use cancelQueries to prevent race conditions

**Cache Management**
- NEVER use setQueryData without structural comparison - bypasses referential equality optimizations
- NEVER forget to handle hydration mismatches - server and client may produce different data

### Mutation Pattern Selection

Choose update strategy based on operation risk:

| Scenario | Pattern | When to Use |
|----------|---------|-------------|
| Form submission | Pessimistic | Multi-step forms, server validation required |
| Toggle/checkbox | Optimistic | Binary state changes, low latency required |
| Drag and drop | Optimistic | Immediate visual feedback essential |
| Batch operations | Pessimistic | Multiple items, partial failures possible |
| Life-critical ops | Pessimistic | Medical, financial, safety-critical systems |
| Audit trail required | Pessimistic | Compliance systems, logged events |

### Observer Count Thresholds

Monitor observer counts in TanStack Query DevTools:

| Observer Count | Performance Impact | Action Required |
|----------------|-------------------|------------------|
| 1-5 | Negligible | None |
| 6-20 | Minimal | Monitor, no immediate action |
| 21-50 | Noticeable on updates | Consider hoisting queries to parent |
| 51-100 | Significant overhead | Refactor: hoist queries or use select |
| 100+ | Critical impact | Immediate refactor required |

## Examples

### Example: QueryClient Factory Pattern

```typescript
// configs/query-client/index.ts
import { QueryClient, isServer } from '@tanstack/react-query';

function makeQueryClient() {
  return new QueryClient({
    defaultOptions: {
      queries: {
        staleTime: 1000 * 20, // 20 seconds
        gcTime: 1000 * 60 * 2, // 2 minutes
        retry: 3,
        refetchOnWindowFocus: false,
      },
    },
  });
}

let browserQueryClient: QueryClient | undefined = undefined;

export function getQueryClient() {
  if (isServer) {
    // Server: ALWAYS create a new query client per request
    // CRITICAL: Prevents data leakage between users
    return makeQueryClient();
  } else {
    // Browser: Create a singleton to maintain cache across navigations
    browserQueryClient ??= makeQueryClient();
    return browserQueryClient;
  }
}
```

### Example: Query Key Factory

```typescript
// Hierarchical factory for consistent invalidation
export const trackKeys = {
  all: () => ['tracks'] as const,
  lists: () => [...trackKeys.all(), 'list'] as const,
  list: (filters: string) => [...trackKeys.lists(), filters] as const,
  details: () => [...trackKeys.all(), 'detail'] as const,
  detail: (id: string) => [...trackKeys.details(), id] as const,
};

// Usage in components
function TrackList({ filters }: { filters: string }) {
  const { data } = useQuery({
    queryKey: trackKeys.list(filters),
    queryFn: () => fetchTracks(filters),
  });
}

function TrackDetail({ id }: { id: string }) {
  const { data } = useQuery({
    queryKey: trackKeys.detail(id),
    queryFn: () => fetchTrack(id),
  });
}

// Invalidation after mutations
function useUpdateTrack() {
  const queryClient = useQueryClient();
  
  return useMutation({
    mutationFn: updateTrack,
    onSuccess: (_, variables) => {
      // Invalidate specific track
      queryClient.invalidateQueries({ queryKey: trackKeys.detail(variables.id) });
      // Invalidate all lists
      queryClient.invalidateQueries({ queryKey: trackKeys.lists() });
    },
  });
}
```

### Example: Optimistic Update with Rollback

```typescript
function useToggleFavorite() {
  const queryClient = useQueryClient();
  
  return useMutation({
    mutationFn: (trackId: string) => api.toggleFavorite(trackId),
    
    onMutate: async (trackId) => {
      // Cancel background refetches
      await queryClient.cancelQueries({ queryKey: trackKeys.detail(trackId) });
      
      // Snapshot current data for rollback
      const previous = queryClient.getQueryData(trackKeys.detail(trackId));
      
      // Optimistically update cache
      queryClient.setQueryData(trackKeys.detail(trackId), (old: Track) => ({
        ...old,
        isFavorite: !old.isFavorite,
      }));
      
      // Return context for rollback
      return { previous, trackId };
    },
    
    onError: (err, trackId, context) => {
      // Rollback to snapshot on error
      if (context?.previous) {
        queryClient.setQueryData(trackKeys.detail(trackId), context.previous);
      }
    },
    
    onSettled: (_, __, trackId) => {
      // Always refetch to ensure cache matches server
      queryClient.invalidateQueries({ queryKey: trackKeys.detail(trackId) });
    },
  });
}
```

}

function TrackListGood() {
  const { data: tracks } = useQuery({

    queryFn: fetchAllTracks,
  });
  
  return tracks.map(track => <TrackItem key={track.id} track={track} />);

function TrackItem({ track }: { track: Track }) {
  // No query, just receives data as props

### Example: Server Component Integration

```typescript
// app/tracks/page.tsx (Server Component)
import { HydrationBoundary, dehydrate } from '@tanstack/react-query';
import { getQueryClient } from '@/configs/query-client';
import { TrackList } from './track-list';

export default async function TracksPage() {
  const queryClient = getQueryClient();
  
  // Prefetch on server
  await queryClient.prefetchQuery({
    queryKey: trackKeys.lists(),
    queryFn: fetchTracks,
  });
  
  return (
    <HydrationBoundary state={dehydrate(queryClient)}>
      <TrackList />
    </HydrationBoundary>
  );
}

// app/tracks/track-list.tsx (Client Component)
'use client';

export function TrackList() {
  // Uses prefetched data, no loading state needed
  const { data } = useSuspenseQuery({
    queryKey: trackKeys.lists(),
    queryFn: fetchTracks,
  });
  
  return <div>{/* render tracks */}</div>;
}
```

### Example: Multi-Layer Cache Invalidation

```typescript
// Server-side cache with Next.js use cache
import { cache } from 'react';
import { unstable_cacheTag as cacheTag } from 'next/cache';

export const getTracks = cache(async (filters: string) => {
  'use cache';
  cacheTag(trackKeys.list(filters).join(':'));
  
  return await db.query.tracks.findMany({ where: filters });
});

// Server action for mutations
'use server';
import { revalidateTag } from 'next/cache';
import { updateTag } from 'next/cache';

export async function updateTrack(id: string, data: TrackData) {
  await db.update(tracks).set(data).where(eq(tracks.id, id));
  
  // Invalidate server cache
  updateTag(trackKeys.detail(id).join(':'));
  updateTag(trackKeys.lists().join(':'));
}

// Client-side mutation
'use client';

function useUpdateTrack() {
  const queryClient = useQueryClient();
  
  return useMutation({
    mutationFn: (variables) => updateTrack(variables.id, variables.data),
    onSuccess: (_, variables) => {
      // Invalidate client cache
      queryClient.invalidateQueries({ queryKey: trackKeys.detail(variables.id) });
      queryClient.invalidateQueries({ queryKey: trackKeys.lists() });
    },
  });
}
```

## Common Pitfalls

> **Good to know:** This skill manages server state (API data, database records). For UI state (modals, themes, form drafts), use Zustand or useState instead.

> **Good to know:** Query keys must be JSON-serializable and deterministic. Avoid Date.now(), Math.random(), unsorted arrays, or objects without stable serialization.

> **Good to know:** When using `select` to transform data, extract the selector to a stable module-level variable. Inline functions re-run on every render.

> **Good to know:** `cancelQueries` in `onMutate` is critical for optimistic updates. Background refetches can overwrite your optimistic changes before the mutation completes.

> **Good to know:** HydrationBoundary must wrap client components only. Server components bypass the boundary entirely.

> **Good to know:** For performance debugging, check observer counts first (TanStack Query DevTools), then check data size and update frequency, then profile React components.
