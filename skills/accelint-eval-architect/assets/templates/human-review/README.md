# Evals — __SKILL_NAME__ (human review)

This skill's output is taste-based, so the eval is a structured human-review
checklist, not an automated harness. There is nothing to install and nothing
to run.

## How to review
1. Open `REVIEW-CHECKLIST.md` and make one copy per output reviewed.
2. Confirm the automated gate items passed (if a deterministic harness exists,
   they run there).
3. Work through the judgment items; record reviewer, date, and verdict.
4. Store the filled-in copy where decided in `DESIGN.md` (PR description,
   `evals/reviews/`, …) so reviews accumulate instead of evaporating.

## Don't lose this
The checklist and DESIGN.md are source — commit them.
